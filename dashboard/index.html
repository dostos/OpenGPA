Round notes

Per-round commentary from docs/eval-rounds/*.md — for context behind the numbers above. Newest expanded by default.

  1. Qualified = solved · scorer=file_level · confidence=high. The complement (rescued) is a solve salvaged by the judge or prose scorer — a proxy for false positives in the maintainer-framing pipeline. The BHDR column can shrink even when BHDR solves grow if BHDR-mode rescues collapse into qualified — that's the desired direction.
  2. Paired data: only (round, scenario) combinations where both code_only and with_bhdr ran are used in lift calculations. R16-R18 dropped BHDR on source-less cohorts; those rows are visible in the per-scenario trace but excluded from lift.
  3. Fix-annotated = scenario.md carries a parseable ## Fix YAML block (fix_pr_url, fix_sha, bug_class, files). Required by the file-level scorer — without it the agent's proposed patches can't be compared against ground truth, so verdict orchestration falls through to prose / judge.
  4. With snapshot = scenario references an upstream repo + SHA. The harness clones the framework at the pre-fix state so the agent can read_upstream / grep_upstream across the real codebase, not just the scenario's minimal repro. Snapshot and fix-annotation overlap but aren't equivalent — many snapshot scenarios lack a resolved PR.
  5. Stable-failure = scenario flagged in scenario.yaml: expected_failure as a known non-product failure (e.g. reasoning ceiling at this model tier, mining picked the wrong PR). Counts but excluded from regression-only solve-rate.
