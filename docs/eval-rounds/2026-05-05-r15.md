## Findings

- **The outline command shipped but did nothing measurable.** R14
  with_gla=7, R15 with_gla=7. Tools used 196 → 208 (more, not fewer).
  Output tokens 97k → 100k. Agent isn't using outline first; it
  still calls `read PATH` directly. Either: prompt mention is too
  subtle, or agents trained on Anthropic SDK reflexively reach for
  `read` on paths.
- **Half-revert of budget hint left half the regression in place.**
  R13 → R14 dropped 6 solves total (heavy budget hint in all three
  prompts). R14 → R15 reverted from maintainer only; advisor +
  config_advice still carry the hint. Most R12 cohort scenarios are
  classified `user-config` or `consumer-misuse`, so they hit the
  hint and stop early. This is why R15 didn't recover R13's numbers.
- **`godot_weird_shadow` collapse confirms the diagnosis.** R14
  code_only stopped at 4 tool calls; R15 code_only stopped at 13
  but still failed. The advisor-prompt budget hint discourages
  thorough investigation on scenarios with multi-file fixes.
- **`godot_4_2_world_environment` continues to fail.** 4 rounds in
  a row. Reasoning-shallow problem; outline doesn't help because
  the agent has the wrong starting hypothesis. Forensic doc holds.
- **R15's only clear win**: `godot_web_images` flipped solved in
  with_gla (was failed in R14). Still 1 of 14 — within stochastic
  noise.

## System-improvement audit (Musk 5-step)

**1. Requirements less dumb.**

Three requirements re-examined:

- **"Run with_gla AND code_only on every scenario."** Owner: original
  eval design (me). Still load-bearing? **No.** For mined scenarios
  with no source_path (13 of 14 in R12 cohort), `BHDR_FRAME_ID` is
  never set, so `gpa frames overview` returns nothing. with_gla is
  literally code_only with extra prompt. The R14 browser-tier gate
  was a step-3 patch on this dumb requirement. **Action for R16**:
  delete with_gla runs for source-less scenarios entirely.
- **"Budget-hint guidance applies to all bug classes."** Owner: me
  (R14). Still load-bearing? **No.** Maintainer scenarios genuinely
  need 15-25 tool calls; advisor/config scenarios with multi-file
  fixes (mining mis-classified) also need it. **Action for R16**:
  delete the budget hint from advisor + config_advice too.
- **"Legacy DiagnosisScorer maintains v1↔v2 telemetry."** Owner:
  comment in `eval-results.md`, no living person. Still load-bearing?
  **No.** Has not been compared in 4 rounds. **Action for R16**:
  delete `DiagnosisScorer` and its `correct_diagnosis` /
  `correct_fix` fields on `EvalResult`.

## Open backlog (R16)

The audit shifts the next round from "ship more features" to "delete
half the system." Three concrete deletions:

- **R16-P0 — delete with_gla runs for source-less scenarios.** When
  `scenario.source_path` is empty, the harness should refuse to
  schedule a with_gla run. Saves ~50% of cohort runtime + judge cost
  on the R12 cohort. The R14 browser-tier gate becomes redundant
  for these (still useful for source-having browser-tier scenarios,
  if any exist).
- **R16-P1 — delete budget hint from advisor + config_advice.** R14
  (full hint) regressed −6; R15 (half-revert) recovered 0. Full
  revert expected to restore R13's numbers.
- **R16-P2 — delete legacy DiagnosisScorer + correct_diagnosis/
  correct_fix fields.** Unused for 4 rounds; their persistence
  bloats the result schema and the harness initialization.

If R16's audit produces no further deletions, the eval-pipeline +
prompt surface is approximately right. Subsequent rounds focus on
real product capability (WebGL backend, framework metadata sidecar,
new GL function intercepts).

  agents need EXPLICIT guidance to use a new tool, not mere mention.

  Step 2: read --lines around the suspect function."
- `godot_weird_shadow` and `godot_perf_on_android` continue to

  to record both PRs as alternatives, or accept the loss.
