## System-improvement audit (Musk 5-step)

**1. Requirements less dumb.**

- "Outline-first triage is universally faster." Owner: me, R15.
  Still load-bearing? **No** — it's faster only when a file is huge
  AND the agent's hypothesis is roughly right. For godot scenarios
  where the bug isn't where the agent first looks, outline-first
  produces a confident wrong diagnosis with too-few tool calls.
  Outline as a tool is fine; outline as a recipe was wrong.

**2. Deletions.**

- `DiagnosisScorer` + `correct_*` + `RunEval` chain (`346e9e6`).
  Dead code under the verdict orchestrator since R12c-era.
- "Prefer outline → read --lines over: read PATH" recipe (`2582df4`).
  R16-rerun forensics directly identified this as the regression
  cause; deletion recovered 2 of 3 target scenarios.

**3. Simplifications.**

- `EvalResult` schema: 2 fewer fields (correct_diagnosis,
  correct_fix). `from_dict` tolerates legacy result files by
  silently dropping the deleted keys.
- `gpa.eval.curation.classify._solved()` helper: single source of
  truth for the "did this result solve?" question — reads
  `verdict["solved"]`. Replaced 4 call-sites that read
  `correct_diagnosis`.

**4. Cycle-time improvements.**

(none this round — but the with_gla skip from R16 keeps cutting
~50% of cohort time)
